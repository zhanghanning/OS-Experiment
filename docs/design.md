# 小型操作系统仿真系统设计说明

## 1. 总体设计思想

本系统不是一个真正能启动的操作系统内核，而是一个操作系统核心机制的仿真系统。它把处理机调度和虚拟存储管理放在同一个程序框架中，用程序状态变化来展示操作系统如何分配 CPU 时间和内存空间。

系统分为四个主要模块：

```text
MiniOS
├── Process / PCB：描述进程状态
├── RoundRobinScheduler：负责处理机调度
├── LRUMemoryManager：负责页面访问和 LRU 页面置换
└── ConsoleUI：负责菜单、输入和运行状态输出
```

## 2. 实验一：处理机调度

实验一按照指导书要求使用时间片轮转法。

每个进程用 PCB 表示，PCB 中保存：

- 进程名
- 要求运行时间
- 已运行时间
- 当前状态
- next 指针字段
- 页访问序列

调度流程如下：

```text
初始化 PCB
↓
建立就绪队列
↓
从队首取出一个进程
↓
该进程运行一个时间片
↓
已运行时间 + 1
↓
如果进程结束，则从队列删除
如果未结束，则重新进入队尾
↓
重复直到所有进程结束
```

这里的 `nextIndex` 字段用于模拟指导书中的“指针”。程序中实际使用 `deque<int>` 维护就绪队列，这样更安全，也更容易输出队列变化。

## 3. 实验二：LRU 页面置换

实验二使用 LRU 作为页面置换算法。

LRU 的核心思想是：

> 当物理内存已经满了，并且新访问的页不在内存中时，淘汰最长时间没有被访问过的页面。

程序中使用 `lastUsedTime` 记录每个页最近一次被访问的时间。每访问一个页面，系统时钟加 1。

访问流程如下：

```text
访问页 page
↓
判断 page 是否在物理块中
├── 在：命中，更新最近访问时间
└── 不在：缺页
    ├── 物理块未满：直接装入
    └── 物理块已满：淘汰最近最少使用页，再装入新页
↓
输出物理块状态、缺页次数、置换次数和命中率
```

## 4. 指令序列和页地址流

程序生成 320 条指令访问序列。生成规则模拟局部性原理：

- 约 50% 是顺序执行。
- 约 25% 访问低地址部分。
- 约 25% 访问高地址部分。

页号转换公式：

```text
页号 = 指令地址 / 10
```

例如：

```text
0 ~ 9     -> 第 0 页
10 ~ 19   -> 第 1 页
310 ~ 319 -> 第 31 页
```

## 5. 综合运行设计

综合运行不是简单地把两个实验放在一个菜单里，而是把它们连接成一个仿真流程：

```text
调度器选中一个进程
↓
进程运行一个时间片
↓
进程访问一个页面
↓
内存管理模块判断是否命中
↓
如果缺页，使用 LRU 置换页面
↓
输出 PCB 状态和物理块状态
```

这样可以体现操作系统中的两个核心问题：

- CPU 时间资源如何分配。
- 内存空间资源如何分配和回收。

## 6. 可扩展方向

当前系统已经实现实验要求。后续可以继续扩展：

1. 增加 FIFO、OPT、LFU 等页面置换算法。
2. 增加 FCFS、SJF、优先级调度等处理机调度算法。
3. 增加动态分区分配和回收。
4. 增加图形界面或日志文件输出。
5. 为每个进程设置独立的虚拟地址空间。
